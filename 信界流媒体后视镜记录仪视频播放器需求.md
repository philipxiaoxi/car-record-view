# 信界流媒体后视镜记录仪视频播放器PRD文档

## 项目定位

信界流媒体后视镜记录仪录制后，视频文件为 ts，首先是不方便播放，另外其前录制视频文件放置在 F，后录制视频放在 R，无法同时播放去观看，无法连续自动播放，导致体验比较差，急需一款 web 应用能够兼容手机移动端、PC 端播放的网页应用。

## 核心原则
要支持移动端、PC 端，响应式设计。

## 页面构成

### 页面一：视频列表

- 视频列表应该支持响应式，使用 flex 布局实现。
- 每个元素有视频的前视封面，封面的下方显示视频录制时间，以年月日时分秒显示。
- 支持分页，一页 50 条。
- 点击某条记录进入播放页。
- 从播放页返回时，能够返回对应的滚动位置高度和对应的页码。

### 页面二：视频播放

- 支持同时播放同一时间段的前视和后视两个视频，视频要同步播放
- 两个视频必须同步：播放、暂停、进度跳转、倍速变化，前后视频保持一致
- 移动端上下布局，PC 端左右布局，尽可能的显示更多视频内容
- 播放控制：播放/暂停、进度条拖动
- 倍速：0.5x、1x、1.5x、2x、4x
- 上一集/下一集：切换到相邻时间段的视频

## 视频存放目录规则

指定一个根目录，可以写到配置里面，该目录下有两个子文件夹：

```
{用户指定目录}/
├── F/          # 前视摄像头视频
│   ├── V20250323-195526F.ts
│   ├── V20250323-200115F.ts
│   └── ...
└── R/          # 后视摄像头视频
    ├── V20250323-195526R.ts
    ├── V20250323-200115R.ts
    └── ...
```

- `F` 文件夹存放所有前视视频
- `R` 文件夹存放所有后视视频
- 文件命名不变，仍为 `V{YYYYMMDD}-{HHMMSS}{F|R}.ts`


```
V{YYYYMMDD}-{HHMMSS}{F|R}.ts
```

- `YYYYMMDD` — 录制日期
- `HHMMSS` — 录制时间
- `F` = 前视摄像头，`R` = 后视摄像头
- 扩展名 `.ts`（MPEG-TS 格式，H.264 编码）

**示例：**

- `V20250323-195526F.ts` — 2025年3月23日 19:55:26 的前视视频
- `V20250323-195526R.ts` — 同一时间的后视视频

通过文件名中的时间戳（`YYYYMMDD-HHMMSS`）进行配对。同一时间戳的 F 文件和 R 文件构成一组，播放时需要同时加载。可能存在只有前视或只有后视的情况（即某时间戳只在 F 或 R 文件夹中存在）。
